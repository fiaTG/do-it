# Architecture Decision Records (ADRs)

Ein **ADR** hält eine einzelne, wichtige Architektur-Entscheidung fest: *was*
entschieden wurde, *warum*, und welche *Konsequenzen* das hat. So bleibt später
nachvollziehbar, warum das Projekt so aufgebaut ist – auch für dich selbst in
einem Jahr oder für jeden, der das Projekt anschaut.

## Format

Jedes ADR folgt [`0000-template.md`](0000-template.md):

- **Status** – Vorgeschlagen · Akzeptiert · Abgelöst durch ADR-XXXX
- **Kontext** – Ausgangslage und Problem
- **Entscheidung** – was konkret beschlossen wurde
- **Konsequenzen** – positive wie negative Folgen
- **Alternativen** – was sonst erwogen und warum verworfen wurde

## Regeln

- ADRs werden **nicht** nachträglich umgeschrieben. Ändert sich eine
  Entscheidung, entsteht ein **neues** ADR, das das alte „ablöst".
- Nummerierung fortlaufend, vierstellig.
